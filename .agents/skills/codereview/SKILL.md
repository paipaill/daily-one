# Code Review Skill

这个技能用于审查代码质量并提供改进建议。在执行此任务时，你需要主动从以下维度切入：

## 1. 结构与可读性审查
- 检查 HTML 结构是否合理且语义化，CSS 命名是否规范（如是否可采用 BEM 等规范化方案简化样式的复杂度）。
- 检查 JavaScript 变量、函数命名是否达意，核心逻辑的封装是否足够内聚。
- 判断当前模块或是否存在“过长函数”（如当前的 `index.html` 包含所有代码，是否需要在未来重构拆分开 API请求、UI渲染、本地存储等独立模块）。

## 2. 逻辑与边界情况审查
- 深入检查 `localStorage` 相关的读写与序列化逻辑，是否存在配额超限或数据损坏的潜在隐患。
- 严格分析 API 请求失败、超时及网络异常时的降级处理（Fallback 机制）是否健壮，是否对用户透明。
- 检查与时间相关逻辑的健壮性（如跨天边界刷新、倒计时精度误差补偿、客户端时区问题导致的数据过期判定）。

## 3. 性能与代码健康度分析
- 审查是否存在内存泄漏的风险（如频繁绑定的事件监听器未被移除、定时器/Canvas资源未得到妥善释放）。
- 评估是否有不必要的重绘（Repaint）或高频 DOM 重排（Reflow）操作。
- 评估图片及多媒体资源的加载开销，如 Unsplash 图片参数调节与响应式加载优化。

## 4. 提供反馈的标准动作
- **精准定位问题**：明确指出需要优化的具体文件及其起止行号。
- **深度剖析原因**：用简练的专业语言解释出现该问题的影响及可能后果。
- **提供直接可执行的 Code 片段**：直接给到最佳实践的替换代码，并详细解释重构方案。
